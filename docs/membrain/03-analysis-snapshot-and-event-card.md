# 分析快照与事件影响卡片设计

## 目标
让项目逐步具备“可回看、可比较、可追踪”的研究能力，而不是只有当前这一轮回答。

## 一、分析快照卡片

### 定义
分析快照是一次分析结果的结构化摘要，不保存整段长对话，而是保存最值得下次回忆的内容。

### 适用场景
- 用户在聊天里问了一只股票
- 用户看完自选股 AI 分析后，想沉淀一个结论
- 系统自动保存最近一次重点判断

### 建议字段
- 标题
- 摘要
- 看多要点
- 看空要点
- 关键变化
- 置信度
- 来源类型
- 创建时间

### 页面展示
在自选页或独立记忆页中展示最近快照列表：
- 每张卡片只显示标题、摘要、时间
- 点开可展开 bull / bear / key change

### 价值
- 下次问同一只股票时可以直接参考
- 用户可以回看最近几次判断
- 后续可扩展成复盘系统

## 二、事件影响卡片

### 定义
事件影响卡片不是“有新闻”，而是“这条新闻 / 公告对原逻辑意味着什么”。

### 适用场景
- 公告
- 财报
- 龙虎榜
- 价格异动
- 行业新闻

### 建议字段
- 事件类型
- 事件摘要
- 对原逻辑的影响
- 哪些点被强化
- 哪些点被削弱
- 后续观察点
- 事件时间

### 输出风格
推荐输出：
- 本次事件强化了什么
- 本次事件削弱了什么
- 对当前关注逻辑是偏正面、偏负面还是中性
- 接下来该继续观察什么

避免输出：
- 绝对结论式买卖建议

## 三、MVP 取舍
本次代码实现优先落：
- 分析快照
- 快照列表查看
- 聊天读取最近快照

事件影响卡片先以文档形式固化设计，代码阶段只预留接口能力，不做完整触发系统。

## 四、后续路线
- 自动从新闻 / 公告生成事件影响卡片
- 在自选页显示“最新事件对逻辑的影响”
- 与预警中心结合，形成主动提醒
