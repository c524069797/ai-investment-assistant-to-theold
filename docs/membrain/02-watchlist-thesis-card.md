# 自选逻辑卡片设计

## 目标
把当前“只有股票名和 AI 分析结果”的自选页，升级成“带关注逻辑的研究卡片”。

## 为什么要做
目前自选页的问题：
- 只能看到这只股票在列表里
- 看不到用户为什么加自选
- AI 分析是即时的，但没有用户自己的判断背景
- 下一次分析无法判断“逻辑是否变化”

所以需要补一张自选逻辑卡片，回答 5 个问题：
1. 为什么关注
2. 看多依据是什么
3. 看空 / 风险点是什么
4. 后续要盯哪些信号
5. 哪些条件出现后，这条逻辑就失效

## 卡片字段
- 股票名称 / 代码
- 关注原因
- 看多要点
- 风险点
- 观察信号
- 失效条件
- 最近判断
- 最近更新时间

## 页面交互

### 展示态
在自选股卡片下方增加一个 MemBrain 卡片块：
- 标题：`关注逻辑`
- 内容摘要：优先展示“关注原因”和“最近判断”
- 操作：`编辑逻辑`、`查看快照`

### 编辑态
建议使用 Drawer 或 Modal：
- 文本域：关注原因
- 标签输入：看多要点
- 标签输入：风险点
- 标签输入：观察信号
- 标签输入：失效条件
- 文本域：最近判断
- 保存按钮

## 文案风格
强调研究和跟踪，而不是买卖建议。

推荐文案：
- 关注原因
- 当前主要依据
- 需要继续验证的点
- 风险与失效条件
- 最近一次判断

不推荐文案：
- 买入理由
- 卖出建议
- 必涨逻辑

## 与聊天联动
当用户从自选页跳转到聊天页时，系统应自动注入：
- 股票代码
- 股票名称
- 当前逻辑卡片摘要

从而让 AI 在第一轮回答里就知道：
- 用户为什么关注它
- 应该重点围绕哪些点来分析

## 数据结构建议
```json
{
  "symbol": "600519",
  "market": 1,
  "watchReason": "白酒龙头，关注估值切换与消费复苏",
  "bullPoints": ["现金流稳健", "品牌壁垒强"],
  "bearPoints": ["估值偏高", "消费恢复节奏需验证"],
  "watchSignals": ["季度业绩", "批价", "消费政策"],
  "invalidationConditions": ["连续两个季度需求弱于预期"],
  "lastJudgement": "中期逻辑仍在，但短期更看估值消化情况"
}
```

## MVP 最小版本
第一版只做：
- 单只股票一张逻辑卡片
- 自选页可查看 / 编辑
- 聊天可读取
- 不做版本对比
- 不做复杂时间线

## 后续增强
- 逻辑版本历史
- 逻辑变化 diff
- 绑定具体新闻 / 公告来源
- 自动生成“逻辑变化提醒”
